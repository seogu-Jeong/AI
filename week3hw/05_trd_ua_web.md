# TRD: Technical Architecture for Universal Approximation Lab

## 1. Architecture Overview
The Universal Approximation Lab is an interactive web application that leverages client-side computation to perform function approximation in real-time. It follows a **Modular Monolith** architecture on the frontend to ensure low latency and high interactivity.

## 2. Technology Stack
- **Framework:** Next.js 14 (App Router) for the main application structure.
- **State Management:** `Recoil` or `Zustand` for managing the target function data and neuron weights.
- **Graphic Engine:** `Konva.js` or `Canvas API` for the interactive "Draw" mode.
- **Plotting Library:** `Plotly.js` or `ECharts` for high-performance line charts.
- **Numerical Library:** `numjs` or `Stdlib.js` for matrix operations.

## 3. Core Engine Implementation

### 3.1 Approximation Logic (Ported from `05_universal_approximation.py`)
The engine will implement a single-hidden-layer network:
$$ y(x) = \sum_{i=1}^{N} w_i \cdot \phi(v_i x + b_i) + \text{out\_bias} $$
Where:
- $v_i$ are input weights (scalars).
- $b_i$ are hidden biases.
- $w_i$ are output weights.
- $\phi$ is the activation function (Tanh/Sigmoid).

### 3.2 Real-time Training Strategy
To allow the "Neuron Slider" to update the graph instantly, we cannot perform full gradient descent on every slider move.
- **Strategy A (Look-up Table):** Pre-calculate optimal weights for preset functions at $N=\{1..100\}$ and interpolate.
- **Strategy B (Random Vector Functional Link - RVFL):** Keep $v_i$ and $b_i$ fixed (randomly initialized) and only solve for output weights $w_i$ using **Least Squares Regression**. This is extremely fast (matrix inversion) and satisfies the theorem's requirement.

## 4. Input Handling: "Draw Mode"
- **Canvas Capture:** Capture mouse coordinates as the user draws.
- **Resampling:** Convert raw points into a fixed-size array of 100 points $\{x, y\}$ where $x$ is equally spaced from 0 to 1.
- **Smoothing:** Apply a Gaussian filter to the drawn points to ensure the function is continuous (as required by the theorem).

## 6. Performance Optimization

### 6.1 WebGL Plotting
For rendering up to 100 hidden neurons' individual contributions (bumps) plus the main plot, we will use a WebGL-backed library like `uPlot` to ensure smooth interaction even on low-end devices.

### 6.2 Memoization
Memoize the forward pass results for the "True Function" so it doesn't recalculate unless the user draws a new one.

## 8. Mathematical Reference
The implementation must match the logic in `05_universal_approximation.py`:
- **Tanh:** `np.tanh(x)`
- **Sigmoid:** `1 / (1 + Math.exp(-x))`
- **MSE Calculation:** `mean((y_pred - y_true)**2)`

## 9. Testing & Quality Assurance
- **Mathematical Accuracy:** Compare the MSE generated by the JS engine with the output of `05_universal_approximation.py` for the Sine wave preset.
- **Touch Interaction:** Verify the "Draw Mode" works correctly on iOS/Android browsers using Touch Events.
- **Unit Tests:** Test the Matrix inversion logic (for RVFL) against edge cases (e.g., singular matrices).

## 10. Deployment Strategy
- **Platform:** Vercel.
- **CI/CD:** Automatic deployment on push to `main` branch.
- **Environment Variables:** None required (fully client-side).

## 11. Security Considerations
- **Content Security Policy (CSP):** Ensure the plotting library doesn't violate CSP rules.
- **Input Validation:** Sanitize user-drawn points to ensure they are within the expected bounds $[0, 1]$.
